# 📄 OCR-Enhanced Word Document Generation from Images

Welcome to the repository for the **OCR-Enhanced Word Document Generation** project. This project provides a highly robust, two-stage AI pipeline designed to convert unstructured, image-based documents (like scanned papers, whiteboard photos, or screenshots) into richly formatted and fully editable Microsoft Word (`.docx`) files.

This project was developed during an internship at **APT Software Avenues Pvt. Ltd.**

---

## 🌟 Project Overview

Traditional Optical Character Recognition (OCR) systems often extract raw text but fail to preserve the structural layout, tables, code blocks, or specific formatting of the original document. 

This project solves that problem by bridging advanced deep-learning-based Document Layout Analysis (DLA) with a highly customized document reconstruction engine. By using Markdown as an intermediate representation, the pipeline intelligently preserves the visual and semantic integrity of complex documents.

---

## 🚀 Key Features

* **Advanced Layout Analysis:** Utilizes `PaddleOCR` (PP-StructureV3) to accurately detect and segment headings, paragraphs, lists, and images.
* **Complex Table Reconstruction:** Accurately extracts tabular data and preserves horizontal cell merging (`colspan`) when writing to the Word document.
* **Syntax-Highlighted Code Blocks:** Automatically detects code snippets and applies server-side syntax highlighting using `Pygments` directly inside the `.docx` file.
* **Dynamic Mermaid Diagrams:** Automatically renders `mermaid.js` text syntax into visual diagrams via the `mermaid.ink` API and embeds them as images.
* **Intelligent Formatting:** Converts Markdown into native Microsoft Word styles (H1-H6, blockquotes, inline code, bold/italics) for immediate editability.
* **Image & Hyperlink Support:** Seamlessly downloads and embeds images from URLs and creates active hyperlinks within the document.

---

## 🧠 System Architecture

The pipeline operates in two distinct stages:

### Stage 1: Image to Structured Markdown
Powered by **PaddleOCR's PP-StructureV3**.
1. **Document Unwarping & Orientation:** Corrects rotated or curved document images.
2. **Feature Extraction:** Identifies bounding boxes for text, tables, and images.
3. **Markdown Generation:** Assembles the extracted text and layout data into a clean, human-readable `.md` file, preserving the logical flow.

### Stage 2: Markdown to DOCX (Custom Converter)
Powered by a custom Python class: `MarkdownToDocxConverter`.
1. **HTML Parsing:** Converts the Markdown into an HTML tree and parses it using `BeautifulSoup4` for robust nested-element handling.
2. **Style Mapping:** Applies custom Word styles to the parsed elements using `python-docx`.
3. **Advanced Rendering:** Handles the specialized rendering of syntax highlighting and diagram generation before outputting the final, high-fidelity `.docx` file.

## 🛠️ Technologies & Libraries

* **Core Language:** Python 3
* **Document Analysis:** `paddleocr` (PP-StructureV3), `paddlepaddle-gpu` / `torch`
* **Document Generation:** `python-docx`
* **Parsing & Formatting:** `markdown`, `BeautifulSoup4`, `Pygments`
* **Web Integration:** `requests` (for image fetching and Mermaid rendering)

## ⚙️ Setup & Installation

To run this pipeline locally, you will need to install the required dependencies. A GPU environment (like Google Colab with an NVIDIA T4) is highly recommended for the PaddleOCR stage.

```bash
# Install PaddlePaddle (GPU version recommended)
pip install paddlepaddle-gpu

# Install PaddleOCR and layout analysis tools
pip install "paddleocr>=2.0.1"

# Install document generation and parsing libraries
pip install python-docx markdown beautifulsoup4 Pygments requests
```

## ⚠️ Known Limitations

While highly accurate, the system currently has a few constraints:
* **Vertical Table Merging:** Due to limitations in the `python-docx` library, complex vertical cell merging (`rowspan`) is not fully supported visually, though the data is preserved in the primary cell.
* **Editable Equations:** Mathematical formulas are currently extracted and embedded as images, not as editable LaTeX/Word math objects.
* **Local Image Paths:** The converter currently relies on URL-based images. Local image paths generated by PaddleOCR require manual resolution before conversion.
